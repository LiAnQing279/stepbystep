用户想要理解一个推理请求在系统中的完整生命周期。

请根据用户指定的场景（框架、模型、并行策略等），追踪一个请求从到达到返回的全路径：

1. **请求到达**：调度层如何接收和排队？
2. **Batch 组装**：如何决定和哪些请求一起处理？
3. **Prefill 阶段**：输入 token 如何被处理？涉及哪些 Kernel？
4. **Decode 阶段**：逐 token 生成时，计算和访存的瓶颈在哪？
5. **KV Cache**：缓存如何分配、管理、淘汰？
6. **并行切分**：如果有多卡，数据如何在卡间流动？
7. **通信开销**：哪些步骤需要卡间同步？延迟多大？
8. **返回结果**：streaming 如何实现？

对于每一步，标注：
- 主要耗时在哪
- 常见优化手段
- 可能的瓶颈

参考：docs/inference-system-map.md

用户指定的场景：$ARGUMENTS
