用户学到了新东西，想要记录下来。

请帮助用户整理笔记并保存到 `notes/` 目录：

1. 先读取用户提供的内容（可能是口述、粘贴的文字、或零散要点）。

2. 整理为以下格式的 Markdown 文件：

```markdown
# [技术名称/主题]

- **所属层级**：
- **解决的瓶颈**：
- **为什么现在重要**：

## 核心要点

[整理后的内容]

## 和系统地图的关联

[与其他层的交互、依赖关系]

## 遗留问题

[还没搞清楚的点，下次深入的方向]
```

3. 文件保存到 `notes/` 目录，文件名用 `YYYY-MM-DD-主题.md` 格式。

4. 如果内容涉及论文或源码阅读，改为存入 `readings/` 目录。

用户要记录的内容：$ARGUMENTS
