用户描述了一个推理部署场景或性能问题，想要诊断瓶颈。

请按以下思路分析：

1. **场景理解**：复述用户的部署配置（模型、硬件、并行策略、负载特征）。

2. **逐层排查**：对系统地图的每一层判断是否可能是瓶颈：
   - 模型结构层：模型的计算/访存比如何？是 compute-bound 还是 memory-bound？
   - Kernel 层：当前 Kernel 是否充分利用了硬件？有无已知更优实现？
   - KV Cache 层：显存是否足够？有无 Cache 淘汰导致重算？
   - 并行层：切分策略是否合理？有无负载不均？
   - 通信层：通信量和带宽是否匹配？是否存在不必要的同步？
   - 调度层：Batch 策略是否合理？是否有请求饿死或气泡？
   - 硬件层：硬件利用率如何？是否有明显的空闲资源？

3. **瓶颈定位**：最可能的瓶颈在哪 1-2 层？为什么？

4. **优化建议**：针对定位到的瓶颈，给出 2-3 个可行的优化方向，按投入产出比排序。

5. **验证方法**：如何用 Profile 工具验证你的判断？该看哪些指标？

参考：docs/inference-system-map.md
